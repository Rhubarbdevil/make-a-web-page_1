Lesson 1.1

The Web (i.e. the Internet)

The web consists of:
Servers --> hold the content (HTML documents)
HTTP requests --> sent from browser to server to get the correct HTML docs
Browsers --> interpret the HTML docs for a user's computer
Computers --> where people interact with the internet (via browsers)

This video has a good explanation.

HTML

HTML stands for Hypertext Markup Language. While there are many types of documents (text, images, videos, etc.) on the web, HTML docs are the main type of document making up the internet.

HTML is made up of:
Text content (what you see on webpages)
Markup (what the text looks like - its formatting)
References to other documents (e.g. images and videos)
Links (to other webpages)

Markup is made of tags. Tags are indicated by angle brackets.
An opening tag + contents + closing tag is an element:

element = <tag> contents </tag>

Attributes go inside tags and assign the tag a value. For instance, <a> is the anchor tag. If you add the href attribute, and assign a web address, you have created a link. The text of the link will be whatever is in the contents of the tag.

IMAGE tags (img) are an example of void tags. That means they don't have closing tags. Void tags have no content.

Image tags have two attributes, the source (src) -- a web address where the image is stored -- and an alternate (alt), which is what will be shown in the event of a broken request (usually "text").

HTML treats all whitespace as a single space. That means tags are necessary to create formatting and whitespace.

<br> </br> is the tag for a line break
<p> </p> is the tag for a paragraph

There are two different types of line/paragraph tag, because they are functionally different. br is an inline tag -- it just ends the line of text. p is a block tag. Block tags create a "box" around the contents of the tag.

HTML docs generally consist of a HEAD and a BODY. The head contains metadata about the doc, while the body contains the actual content (what will appear on the web).

Lesson 1.2

Structure and Style

HTML and CSS are both languages. HTML files are the structure of a website, while CSS files provide the style.

Analogy: HTML is like the walls, floors and ceilings, while CSS is like the color of the walls and how it's decorated.

HTML docs have a "tree-like" structure. This structure is the Document Object Model, or DOM.

HTML turns tags into elements in the DOM.

Webpages can be broken down into boxes. One of the first steps of analyzing a webpage is to "boxify" it, or identify all the boxes used or planned to be used.

Lesson 1.3

General Note

You must always use <!DOCTYPE html> at the top of your document, before the <html> tag.

Styling and CSS

CSS Controls the Style of HTML

CSS stands for Cascading Style Sheets. "Cascading" because the computer looks down through all the rules until it finds the one most specific to that element. That rule then gets applied.

How to Use CSS

There are three ways to add CSS to your HTML doc:

1. You can write CSS in the <head> of your document. To do this, you add a <style> </style> element to your document above the <html>.
2. You can create a separate CSS file and <link> it to the HTML document in order to add style.
3. You can write your style inline with your HTML

Why #3 Above is a BAD IDEA

Repetition. If you write your style inline with your HTML, you will have to potentially add the same style/code over and over and over again. No one wants to do that, and no one respects coders who do that. The great thing about computers is they let you automate tedious tasks. So, instead of re-writing the same style code over and over, you can simply assign that style to a selector (an element type or a class that you assign to an element.) The computer will then style any instance of that selector the same way.

Inheritance

Inheritance is a key feature of CSS. Any styling assigned an element is "inherited" by any element contained inside the styled element. For instance, if <h1> is styled to be blue, and the element <em>for example</em> is inside an <h1> heading, if no other styling is assigned to <em> elements, it will inherit the blut styling from the heading.

Generally speaking, any text-related properties are inherited, but box-related properties are not.

Clarifying Div and Span

<div> and <span> are both containers. The <div> tag is a block tag, and is used to control layout (think boxes). The <span> tag is inline, and is used to style text (i.e. text decorations and enclosing inline elements).

ID's and Classes

id is an attribute that uniquely identifies an element. class is an attribute that styles things that are similar to each other.

The Box Model

It can be hard to space your boxes correctly, because they have margins, borders and padding around the content. There are two ways to handle box size issues:

1. Set box sizes in terms of percentages, not pixels
2. Set the box-sizing attribute to border-box

Flex Box

Adding the rule display: flex to all elements will override the rule that makes each div take up the width of the page. This allows side-by-side boxes.

Coding Methodolgy

Code, Test, Refine

The basic methodology for writing programs is to write code, test it, then refine. Tips for approaching your web page:

1. Work on the structure and get it right - BOXIFY
2. Work on the tags
3. Work on the size of the boxes
4. Work on the positioning of the boxes
5. Work on the smaller details

To reiterate:

1. Look for natural boxes 2. Look for repeated styles and semantic elements 3. Write the HTML 4. Apply your styles from BIGGEST to smallest 5. Fix the details

Use dev tools during your testing and refining.
